Building Pages

  • Pick your template language of your choice
  • Create a file (/foo.md)
  • Eleventy manages the output (/foo/index.html)
  • You have control over that output (URL, filename, etc)

Template Languages

  • HTML
  • Markdown
  • JavaScript
  • Liquid
  • Nunjucks

Template Languages - MOAR!

  • Handlebars
  • Mustache
  • EJS
  • HAML
  • Pug
  • Custom

Template Languages - EVEN MOAR!

  • Markdown + Liquid
  • Mixed
    • You can use Liquid in one file, EJS in another...
    • Or use EJS inside a Liquid template (as one example)
    Markdown files also support Liquid. So you can make a foo.md, include Liquid code, and it will be preprocessed before the Markdown is converted. If you would rather _another_ language, you can configure 11ty. ALso - you can (in 1.0) use a template in one language, like Liquid, and have a block done in another lang. EJS is a good example of this.

Organizing Content

  • Eleventy's interperting files into static assets
  • You have groups of content that are shared:
    • Blog posts
    • Press releases
    • Documentation
  • Eleventy provides a way to define these groups
  • Eleventy provides a way to work with these groups

Collections

  • Way to group files
  • Tag based
  • Tags defined in front matter
  • Can be dynamic

---
layout: somename
title: Something
tags: post
--- 

Your MD file, Liquid, etc
				


{% for post in collections.post %}
  use the post object here: 
  {{ post.title }} written on {{ post.date }}
{% endfor %}

Content Dates

  • Critical for a LOT of content
  • You can define a date in your front matter
  • Eleventy provides one based on file creation
  • Dates should be a valid YAML date value (spec)
  • date: 2021-04-08
  • Also supports "Last Modified" and "Created" (date: Last Modified)
  • Your filepath (file name or folder) can use a YYYY-MM-DD format.

Configuration

  • CLI lets you modify many things
  • Eleventy looks for: .eleventy.js
  • Lets you configure everything
  • Custom filters, short codes, global data, plugins, and more

.eleventy.js


module.exports = function(eleventyConfig) {
	// various APIs available via eleventyConfig object

	// optional return:
	return {
		dir: {
			input:"my_awesome_code",
			output: "my_site"
		}
	}
};

.eleventy.js


module.exports = function(eleventyConfig) {

	eleventyConfig.addPassthroughCopy("img");
	eleventyConfig.addPassthroughCopy("css");
	eleventyConfig.addPassthroughCopy("scripts/*.js");

};

.eleventy.js


module.exports = function(eleventyConfig) {

	eleventyConfig.addPassthroughCopy("css");
	eleventyConfig.addWatchTarget("./css");

};

.eleventy.js


module.exports = function(eleventyConfig) {

	const english = new Intl.DateTimeFormat('en');

	/*
	I can now do: {{ someDate | dtFormat }}
	*/
	eleventyConfig.addFilter("dtFormat", function(date) {
		return english.format(date);
	});

};

.eleventy.js


module.exports = function(eleventyConfig) {

	eleventyConfig.addGlobalData('site', {
		name:'My Site',
		author:'Raymond Camden'
	});

};
